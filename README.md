# GLAS — AI-модуль обработки обращений (NLP + STT + CV)

## О проекте
**GLAS** — интеллектуальная система, которая помогает жителям быстро и удобно сообщать о проблемах городской среды (освещение, мусор, дорожные ямы, благоустройство и т.д.).  
Этот репозиторий содержит **AI-часть** проекта: обработку обращений с помощью нейросетей для текста, голоса и изображений/видео.

Цель AI-модуля — сделать обработку обращений более быстрой, прозрачной и масштабируемой: автоматически понимать суть обращения, определять приоритет и направлять его в нужную службу.

---

## Что умеет AI-модуль

### 1) NLP (Natural Language Processing)
Модуль обработки текста выполняет:
- **определение категории обращения** (например: освещение / мусор / дорога / благоустройство и т.д.)
- **определение срочности** (`low / medium / high / critical`)
- (опционально) **извлечение сущностей** (адрес, ориентиры, объект), если понадобится на следующих этапах

### 2) STT (Speech-to-Text)
Модуль обработки голоса:ш
- переводит **голосовое обращение в текст**
- передаёт транскрипт в NLP
- возвращает **confidence** распознавания, чтобы система понимала, насколько можно доверять расшифровке

### 3) CV (Computer Vision)
Модуль компьютерного зрения анализирует **фото и видео** и помогает:
- **обнаружить и классифицировать проблему** (например: яма, мусор, повреждение, граффити, неработающий фонарь и т.д.)
- (опционально) **локализовать** проблему на изображении (bounding box / сегментация)
- оценивать **качество входных данных** (размыто, темно, не видно объекта) и выдавать флаг `no_issue / low_quality`

### 4) Итоговое решение и маршрутизация
На основе результатов NLP/STT/CV и метаданных (геолокация, канал, время) система формирует итог:
- финальная **категория**
- финальная **срочность**
- **маршрут** (служба/подразделение/очередь)
- флаги качества: `low_confidence`, `needs_review`, `possible_duplicate`, `low_quality_media`

---

## Общая схема обработки обращения
1. Пользователь отправляет обращение **текстом / голосом / фото / видео**
2. Если вход голосовой → **STT** делает транскрипт
3. **NLP** определяет категорию и срочность по тексту (и/или транскрипту)
4. Если есть фото/видео → **CV** определяет тип проблемы и (опционально) её расположение на кадре
5. Модуль маршрутизации объединяет сигналы и формирует итоговое решение, направляя обращение в нужную службу
